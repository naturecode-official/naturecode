#### v1.3.2 新增功能详情

1. **配置管理系统实现** (`src/utils/config-manager.js`)
   - 全局配置路径：~/.naturecode/config.json

   - 配置合并算法：深度合并，项目配置优先于全局配置
   - 配置验证：版本检查、权限级别验证、数组格式验证
   - 深拷贝机制：避免配置对象污染
   - 文件访问控制接口
   - 操作权限检查接口

2. **文件访问控制系统**
   - 扩展名控制：
     - allowedExtensions：白名单模式
     - deniedExtensions：黑名单模式（优先级更高）
   - 目录控制：
     - allowedDirectories：允许的目录列表
     - deniedDirectories：禁止的目录列表
   - 文件大小限制：maxFileSize（默认 10MB）
   - 符号链接处理：followSymlinks 选项
   - 实时验证：isFileAllowed() 方法

3. **三级权限系统**
   - readonly 模式：
     - allowRead: true
     - allowWrite: false
     - allowDelete: false
     - allowExecute: false
   - restricted 模式：
     - allowRead: true
     - allowWrite: true
     - allowDelete: false
     - allowExecute: false
     - 需要确认：删除、覆写、大文件操作
   - full 模式（默认）：
     - allowRead: true
     - allowWrite: true
     - allowDelete: true
     - allowExecute: false

4. **命令别名系统**
   - 别名存储：config.commands.aliases 对象
   - 别名解析：resolveAlias() 方法
   - 别名管理：添加、删除、列出
   - 作用域支持：项目级和全局级别

5. **配置命令处理器** (`src/utils/config-commands.js`)
   - show：显示完整配置（支持 JSON 格式）
   - get：获取特定配置值（支持点号路径）
   - set：设置配置值（支持类型解析）
   - reset：重置为默认配置（需要确认）
   - permission：设置权限级别
   - alias：别名管理（list/add/remove）
   - whitelist：白名单管理（show/add/remove）
   - blacklist：黑名单管理（show/add/remove）
   - check：验证文件或操作权限

6. **依赖管理功能**（项目管理模块扩展）

     - 生成优先级推荐
     - 提供升级命令建议
   - checkDependencyConflicts()：
     - 检测重复依赖（同时在 dependencies 和 devDependencies）
     - 检测 peer dependency 冲突
     - 检测缺失依赖
